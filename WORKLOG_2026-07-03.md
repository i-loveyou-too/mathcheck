## 1. 서버 / 배포 구조 확인 내용

확인된 운영 구조:

- Nginx
  - `listen 3007`
  - `/` -> `proxy_pass http://127.0.0.1:3001`
  - `/api` -> `proxy_pass http://127.0.0.1:8000/`
- frontend
  - Next.js를 PM2로 실행
  - 정상 구조는 `next start -p 3001`
- backend
  - `8000` 포트 사용

중요 원인 확인:

- `/student/today` 화면이 서버에서 예전 UI로 보이던 문제는 단순 브라우저 캐시 추측이 아니라,
  **서버에 있는 실제 frontend source가 최신 로컬 코드와 달랐던 것**으로 확인됨
- 서버의 다음 두 위치 모두 예전 문자열 기준으로 반영되어 있었음
  - `/home/clover/mathcheck/frontend/app/student/today/page.tsx`
  - `/home/clover/mathcheck/frontend/.next/server/app/student/today/page.js`

즉:

- `npm run build`는 성공했지만
- 성공한 빌드가 최신 로컬 수정본이 아니라, 서버에 남아 있던 예전 source 기준 빌드였음

권장 서버 반영 순서:

```bash
cd /home/clover/mathcheck
git pull

cd /home/clover/mathcheck/frontend
rm -rf .next
npm run build
pm2 restart mathcheck-frontend
pm2 save
```

PM2 설정이 꼬였을 때 재등록 기준:

```bash
pm2 delete mathcheck-frontend
cd /home/clover/mathcheck/frontend
npm run build
pm2 start "npm start -- -p 3001" --name mathcheck-frontend
pm2 save
```

## 3. 관리자 교재 관리 페이지 작업

### 교재 상세가 안 열리는 문제

원인:

- 교재 상세 패널을 열 때 아래 두 API를 `Promise.all`로 함께 기다리고 있었음
  - `/admin/textbooks/{id}`
  - `/admin/textbooks/{id}/assignments`
- 이 중 하나만 실패해도 상세 패널 전체가 비어 보일 수 있는 구조였음

조치:

- 상세 API와 assignments API를 분리
- 상세 API가 성공하면 우선 상세 패널은 열리도록 변경
- assignments API가 실패해도 전체 상세가 안 막히도록 수정
- 상세 API 자체 실패 시 오류 문구를 보여주도록 보완

관련 파일:

- `frontend/app/admin/textbooks-management/page.tsx`

## 4. 관리자 학생 상세의 "배정 교재" 누락 문제

문제:

- 관리자 > 학생 상세에서 "배정 교재" 영역에
  - 전체 공개 교재
  - 특정 학생 배정 교재
  가 모두 보여야 하는데 일부가 안 보였음

원인:

- 학생 상세에서 사용하는 API:
  - `GET /admin/textbooks-for-student/{student_id}`
- 이 API의 백엔드 구현에서 교재 목록을 가져올 때
  **`is_checkable = true` 필터가 들어가 있어서**
  체크리스트형 교재만 내려주고 있었음

영향:

- 비체크형 교재
- 특정 학생에게만 배정된 비체크형 교재
가 학생 상세의 배정 교재 목록에서 누락됨

조치:

- `get_textbooks_for_student_catalog()`에서
  `MathTextbook.is_checkable.is_(True)` 필터 제거
- 이제 프론트가 원래 의도한 대로
  - 체크형 교재
  - 기타 교재
  로 나눠서 보여줄 수 있음

관련 파일:

- `backend/crud.py`

## 5. 교재 시스템 관련 최근 정리

최근 교재 시스템은 다음 흐름으로 정리 중:

- DB 기반 `textbook_key` 도입
- 학생 과목별 교재 리스트를 DB API 기반으로 전환
- 학생 체크리스트 페이지를 동적 route 기반으로 전환
- 관리자 숙제 관리 교재 목록도 DB 기반으로 전환

현재 핵심 포인트:

- 기존 `TEXTBOOK_PROGRESS_CONFIG`는 완전히 삭제하지 않고 fallback 성격으로 유지
- 학생 화면과 관리자 화면이 점점 하드코딩 배열 대신 DB 기반 목록을 사용하도록 이동 중
